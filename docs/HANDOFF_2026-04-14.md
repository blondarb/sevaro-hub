# Handoff тАФ 2026-04-14

## Session Summary

Fixed silent 401 when admins clicked **Save Status** on a feedback session
(`/feedback/[id]`). Reported via screenshot showing
`Failed /api/feedback/28f806тАж?appId=evidence-engine 401` in DevTools console.

## Root Cause

Two compounding issues:

1. **Session detail page had no server-side auth gate.** The Lambda GET
   (`sevaro-feedback-api`) only requires `x-api-key`, not a JWT, so the page
   could fully render for a user whose `id_token` cookie had silently expired.
   The PATCH route (`src/app/api/feedback/[id]/route.ts`) DOES require a valid
   JWT, so the failure only surfaced on Save Status тАФ long after the user
   thought they were "in the dashboard."

2. **`getAuthHeaders()` swallowed refresh failures.** When `getIdToken()`
   returned `null` (id_token expired *and* refresh failed), the helper
   returned headers without an `Authorization` entry instead of throwing. The
   PATCH then went out with no Bearer token, hit the cookie fallback in
   `extractToken`, verified the still-expired cookie, and returned 401 with no
   user-visible message.

## What Changed

### `src/app/feedback/[id]/page.tsx`
- Added server-side auth gate before any data fetching. Reads `id_token`
  cookie via `next/headers#cookies()`, calls `verifyToken()`. If invalid тЖТ
  `redirect('/api/auth/login')`. If valid but not admin тЖТ render
  "Admin access required" message.
- This prevents the page from ever rendering session content (or hitting
  the upstream Lambda) for an unauthenticated/expired session, eliminating
  the "stale tab" failure mode.

### `src/app/feedback/[id]/SessionDetailClient.tsx`
- Removed the silently-failing `getAuthHeaders()` helper.
- `handleStatusUpdate` and `handleDelete` now check the token explicitly:
  - If `getIdToken()` returns `null` тЖТ show "Your session has expired.
    Redirecting to sign inтАж" and bounce to `/api/auth/login` after 1.5s.
  - If the PATCH/DELETE returns `401` тЖТ same recovery flow.
- New `handleAuthExpired` callback centralises the messaging + redirect.
