# Handoff — 2026-03-27

## Session Summary

Redesigned the feedback dashboard to support multi-app navigation with per-app analysis capabilities.

### What was built

1. **Shared app registry** (`src/lib/app-registry.ts`) — Single source of truth for app list used across dashboard tabs, analysis scope selector, and What's New admin page.

2. **Sessions proxy API route** (`src/app/api/feedback/sessions/route.ts`) — Authenticated server-side proxy enabling client-side session fetching (needed because `FEEDBACK_API_URL` is a server-only env var).

3. **Dashboard rewrite** (`src/app/feedback/page.tsx`) — Converted from server component to client component with:
   - Tab bar with red unreviewed-count badges per app
   - Global stats bar (sessions, open, critical, apps)
   - Inline category chips (bugs, suggestions, confusion, praise)
   - App badge hidden when viewing a specific app tab
   - Analyze link passes `?app=` scope param

4. **Analysis page scope selector** (`src/app/feedback/analyze/page.tsx`) — Added app scope dropdown pre-populated from URL `?app` param, Suspense wrapper for `useSearchParams`.

5. **Dual Bedrock analysis prompts** (`src/app/api/feedback/analyze/route.ts`) — Per-app prompt (bugs, pain points, app-specific fixes) vs cross-app prompt (portfolio health, systemic patterns, shared UX issues).

6. **What's New migration** (`src/app/admin/whats-new/page.tsx`) — Replaced local `APP_OPTIONS` with shared registry import.

### Decisions

- **Client-side tabs** (Approach A) — Single fetch, filter in state. Simpler than server-side filtering and sufficient for current data volume.
- **Fixed tab list** — All apps always visible (no dynamic tabs based on data presence).
- **Badge = unreviewed count** — `reviewStatus` missing or `'open'` counts as unreviewed.

### Next Steps

- Smoke test on staging after Amplify deploy
- Consider adding `getAppLabel()` for `affectedApps` display in analysis results (currently shows raw app IDs)
- Future: dynamic tab list based on apps with actual feedback data
