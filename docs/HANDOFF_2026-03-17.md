# Handoff — 2026-03-17

## Session Summary

Built Phase 1 of the "What's New" cross-app update notification system for Sevaro Hub.

### What Was Built

**AWS Infrastructure:**
- DynamoDB table `sevaro-whats-new` (PK: `appId`, SK: `timestamp`, pay-per-request)
- Lambda `sevaro-whats-new-api` (Node 22, `aws-jwt-verify` for Cognito JWT validation)
- API Gateway `5168ofhh8k` (HTTP API with CORS, 4 routes)
- IAM inline policy `dynamodb-whats-new` on `sevaro-lambda-basic` role

**API Endpoints:**
- `GET /whats-new?appId=...&since=...` — public, returns app-specific + "all" entries, strips `createdBy`
- `GET /whats-new/all` — admin-only, returns all entries (used by admin UI)
- `POST /whats-new` — admin-only, creates entry with category/link validation
- `DELETE /whats-new` — admin-only, removes entry by composite key

**Hub Frontend:**
- `src/lib/whats-new-api.ts` — typed API client (getUpdates, getAllUpdates, createUpdate, deleteUpdate)
- `src/components/WhatsNewBadge.tsx` — dropdown badge with unseen count, 1-hour sessionStorage cache, localStorage last-seen tracking
- `src/app/admin/whats-new/page.tsx` — admin CRUD page with stats, app filter, create form, delete
- `src/components/NavBar.tsx` — added WhatsNewBadge + admin "What's New" link
- `lambda/sevaro-whats-new-api/index.mjs` — Lambda source with `aws-jwt-verify`

### Decisions Made

- JWT verification uses `aws-jwt-verify` (proper signature + claims check) rather than lightweight base64 decode — caught by GPT cross-model review
- Public GET endpoint strips `createdBy` field to avoid leaking admin emails
- `GET /whats-new/all` is auth-gated (initially was public — caught by review)
- `link` field validated server-side for http/https scheme only
- Used `lastSeenRef` pattern instead of reading localStorage per-item in render loop

### Cross-Model Review
